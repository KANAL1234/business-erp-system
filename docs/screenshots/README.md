## Instructions for Adding Screenshots

To make your README more impressive for recruiters, add screenshots of your application here.

### Recommended Screenshots

1. **dashboard.png** - Main dashboard with analytics and KPIs
2. **pos-interface.png** - Point of Sale interface showing checkout
3. **inventory-management.png** - Inventory management with stock levels
4. **sales-invoice.png** - Professional sales invoice
5. **accounting-reports.png** - Financial reports (Trial Balance or P&L)
6. **mobile-pwa.png** - Mobile PWA interface on a phone

### How to Capture Screenshots

1. **Run the application**: `npm run dev`
2. **Navigate to each module**: Use the sidebar navigation
3. **Capture screenshots**: Use your OS screenshot tool
   - **macOS**: Cmd + Shift + 4
   - **Windows**: Win + Shift + S
   - **Linux**: PrtScn or Screenshot app
4. **Save to this directory**: Save as PNG files with descriptive names
5. **Update README**: The README already references these screenshots

### Screenshot Guidelines

- **Resolution**: 1920x1080 or higher
- **Format**: PNG for best quality
- **Content**: Show real data (or realistic demo data)
- **Branding**: Include your company/project branding
- **Clean UI**: Hide any sensitive information
- **Annotations**: Optional - add arrows or highlights for key features

### Tips for Great Screenshots

1. **Use realistic data**: Don't show empty tables or "Test Product 1"
2. **Show functionality**: Capture the app in action, not just empty forms
3. **Highlight features**: Show the unique features of your ERP
4. **Mobile screenshots**: Use browser dev tools to simulate mobile devices
5. **Dark mode**: Consider showing both light and dark mode versions

### Tools for Better Screenshots

- **Browser Extensions**: Full Page Screen Capture, Awesome Screenshot
- **Editing**: Use tools like Figma, Photoshop, or online editors to add annotations
- **Mockups**: Use device mockups from [Shots.so](https://shots.so) or [Screely](https://screely.com)
